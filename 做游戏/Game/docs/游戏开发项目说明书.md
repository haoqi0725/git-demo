### 游戏开发项目说明书
1. 项目概述
游戏名称：影子回音（Shadow Echo）

游戏类型：2D平台解谜 + 影子机制

核心玩法：玩家可以按空格生成一个影子，影子会重复玩家过去3秒的移动轨迹。利用影子同时激活机关、压住压力板、引开敌人，从而通过关卡。

开发周期：13周（第3周至第15周）

开发语言：C++17

图形库：Raylib 5.0

开发环境：VS Code + WSL (Ubuntu 22.04) + CMake

版本控制：Git（本地仓库，每周提交）

2. 游戏玩法说明
2.1 基本操作
左右箭头键：控制玩家左右移动。

上箭头键：跳跃（后续实现）。

空格键：在当前玩家位置生成一个影子。

P键：暂停（后续实现）。

2.2 影子机制
影子会延迟3秒重复玩家过去3秒内的移动路径。

每个影子持续存在3秒后消失，可以同时存在多个影子。

影子可以触发机关（如压力板），但不会与敌人发生战斗（敌人可能被影子吸引）。

2.3 机关与敌人（后续迭代）
压力板：玩家或影子站在上面时激活，可以打开门或触发其他机关。

门：初始关闭，被压力板激活后打开。

敌人：简单AI（巡逻、追逐），影子经过时敌人会转向或停止。

移动平台：按固定路径移动，玩家可站在上面。

2.4 胜利条件
通过所有关卡，每关需利用影子解开谜题。

3. 系统架构

模块依赖关系：箭头表示调用方向，双向箭头表示互相调用。核心引擎（M01）负责初始化、主循环，并协调各模块的更新和绘制。

4. 模块划分与职责
模块ID	模块名称	主要职责	主要接口（见头文件）
M01	核心引擎	游戏循环、实体管理、模块初始化/更新/绘制、帧率控制	InitCore(), ShutdownCore(), RunGameLoop()
M02	输入管理	处理键盘输入，转换为动作枚举	GetAction()
M03	物理引擎	重力、移动、AABB碰撞检测、碰撞响应	UpdatePhysics(), CheckCollision(), MoveEntity()
M04	角色状态机	管理玩家状态（站立、跑、跳、下落、影子冷却），接收动作并调用物理	InitPlayer(), UpdatePlayer()
M05	影子记录器	存储玩家过去N帧的状态（位置、动作方向），提供查询接口	InitRecorder(), RecordFrame(), GetPastPosition()
M06	影子实体	创建影子实体，根据影子记录器更新其位置，绘制影子	SpawnShadow(), UpdateShadows(), DrawShadows()
M07	机关系统（逻辑）	处理压力板、门、移动平台等机关的状态变化	UpdateMechanics()
M08	机关系统（渲染）	绘制机关（压力板、门等）	DrawMechanics()
M09	敌人AI	简单敌人（巡逻、追逐），与影子交互	UpdateEnemies()
M10	关卡加载	从JSON文件加载关卡数据，生成静态实体列表	LoadLevel()
M11	渲染基础	绘制所有实体的基本形状（矩形、圆形）	DrawEntity()
M12	动画系统	管理精灵动画（角色、敌人等）	UpdateAnimations(), DrawAnimated()
M13	摄像机	跟随玩家，平滑移动，边界限制	InitCamera(), UpdateCamera(), GetCameraOffset()
M14	UI系统	菜单、HUD（分数、影子状态）、暂停	InitUI(), DrawUI(), HandleMenuInput()
M15	音频系统	播放背景音乐、音效	InitAudio(), PlayMusic(), PlaySound(), StopAllAudio()
每个模块的详细接口请阅读对应头文件（位于 include/ 目录）。

5. 开发规范
5.1 代码风格
命名：

类名：大驼峰，如 Player。

函数名：小驼峰，如 UpdatePlayer。

变量名：小写下划线，如 player_position。

宏/常量：全大写下划线，如 MAX_PLAYERS。

缩进：4个空格，不使用Tab。

注释：

每个头文件中的类、公共函数必须有简要注释（功能、参数、返回值）。

复杂逻辑在cpp中加行注释。

每个文件开头可选包含作者信息和创建日期。

括号：左括号不换行，如 if (condition) {。

5.2 Git 提交规范
每周至少提交3次，每次提交信息格式：[模块ID] 描述，例如 M03: 修复重力计算bug。

每次提交应保证编译通过（至少不影响其他模块）。

禁止直接提交到主分支，统一由教师集成。

5.3 接口规范
所有公共接口必须在头文件中声明，并注释清楚。

禁止修改他人模块的头文件，如需修改必须与教师和相关组员协商。

模块间的数据交换尽量通过函数参数传递，避免直接访问全局变量（全局实体列表 g_entities 除外，它是核心引擎维护的共享数据）。

5.4 测试规范
每个模块应提供简单的测试方式，如打印调试信息、单元测试（可选）。

每周集成后，各模块需验证自己的功能是否正常。

9. 常见问题
9.1 编译时报错找不到Raylib
确认已安装 libraylib-dev。

如果使用预编译包，检查 CMakeLists.txt 中的路径是否正确。

9.2 运行时窗口一闪而过
检查 DISPLAY 环境变量是否正确设置（WSL需运行 X Server）。

运行 xeyes 测试图形是否正常。

9.3 影子未出现
确认按空格键时控制台有输出（可在 M06_ShadowEntity.cpp 中添加 printf 调试）。

检查 UpdateShadows 和 DrawShadows 是否在主循环中被调用。

9.4 中文显示乱码
确认系统已安装中文字体（如 fonts-wqy-microhei）。

检查 M14_UI.cpp 中的字体路径是否正确，且使用了 LoadCodepoints 加载所需字符（已提供示例）。
